# 公开展示版 README 设计规格

## 目标

把项目根目录 `README.md` 从偏发布准备/开发说明的文档，改成适合公开展示的产品介绍页。README 应让第一次访问仓库的普通用户在首屏附近知道：这是什么、能在哪些平台运行、如何下载、使用时有哪些隐私和准确性边界；开发者仍能在后半部分找到构建入口。

## 目标读者

1. 普通使用者：希望下载 Windows 或 Android 版本并快速开始检测。
2. 项目合作方/评审者：希望了解识别流程、数据处理方式和已知限制。
3. 开发者：希望从源码构建客户端、运行服务器或理解项目目录。

## 语言和语气

- 中文为主，使用清晰、克制、可验证的产品文案。
- README 顶部附一段简短英文简介和主要能力关键词，不制作完整英文镜像。
- 不夸大识别准确率，不把辅助识别描述为实验室或监管结论。
- 明确说明 Windows 当前为未签名便携版，Android 要求 Android 14/API 34 及以上。

## 页面结构

1. **首屏信息**
   - 项目名称和一句中文定位。
   - 一句英文简介：面向水样反应管照片的离线辅助识别工具。
   - 官网、GitHub、Release、Android API 等徽章或链接。
   - 一段简短的产品边界说明。

2. **媒体展示区**
   - 保留一个主界面截图位置。
   - 保留一个检测结果截图位置。
   - 保留一个 Android 操作流程截图位置。
   - 保留一个 30～90 秒演示视频位置，演示“选择照片 → 检测 → 查看结果 → 保存/可选上传”。
   - 未提供真实素材前，不引用不存在的图片或视频 URL；用公开可见的素材清单和 HTML 注释说明后续应放什么。

3. **核心能力**
   - Windows 桌面检测。
   - Android 端拍照/相册检测。
   - 离线模型检测和结果复核。
   - 结果本地保存，以及用户确认后的可选同步。
   - 发布服务器、统计和版本化下载。

4. **快速开始与下载**
   - 官网直下链接。
   - GitHub `v1.0.5` Release 链接。
   - Windows 便携 ZIP/EXE 使用说明。
   - Android APK 安装条件。
   - SHA-256 校验入口。

5. **基本使用流程**
   - 准备清晰、光线均匀、无隐私内容的照片。
   - 选择水样类型和检测方式。
   - 检查识别框、标签和置信度。
   - 保存结果；需要时再上传同步。

6. **隐私、安全与准确性**
   - 检测结果和图片会一同上传服务器。
   - 数据通常仅在服务器保存一天，主要用于模型优化和结果统计。
   - 上传前必须自行确认照片不含身份证件、面部、住址、联系方式等个人隐私。
   - 删除申请邮箱：`sunx77@mail2.sysu.edu.cn`。
   - 检测结果不能替代实验室检测、人工复核或监管结论。

7. **已知限制**
   - Windows v1.0.5 当前是未签名便携包，可能出现 SmartScreen“未知发布者”。
   - 当前没有 Inno Setup 安装器；ZIP 内的 EXE 必须与 `_internal`、模型文件和 `release.json` 保持目录结构。
   - 真实设备/干净系统验收、第三方许可证逐项核对和应用商店材料属于后续工作。

8. **技术和开发者信息**
   - 用 `<details>` 折叠构建、目录和服务器说明。
   - 保留 Android/Windows 正式构建命令，但所有域名、Token、密码使用示例占位符。
   - 明确禁止提交 keystore、PFX/PEM、SSH 私钥、数据库、用户照片和构建产物。

9. **许可证和第三方依赖**
   - 指向仓库许可证文件（若当前尚未确定，则保留待补充说明，不擅自声明许可证）。
   - 提醒发布前必须核对模型权重、训练数据、字体、Python/Android 依赖和打包工具的可分发授权。

## 媒体占位约定

README 中使用以下公开可见的占位清单，等素材准备好后再替换成真实媒体：

| 素材 | 建议文件/链接 | 应展示的内容 |
| --- | --- | --- |
| Windows 主界面 | `docs/screenshots/01-main-window.png` | 完整主界面、无真实用户数据 |
| 检测结果页 | `docs/screenshots/02-detection-result.png` | 识别框、标签、置信度和保存入口 |
| Android 流程 | `docs/screenshots/03-android-flow.png` | 选择照片、检测中、结果页三步拼图 |
| 演示视频 | GitHub 上传视频或公开视频链接 | 30～90 秒完整操作流程，不能出现隐私照片或 Token |

## 同步范围

- 更新公开发布仓库 `README.md`。
- 将相同 README 同步到源码工作区，避免两个版本的公开说明分叉。
- 不在本次任务中修改客户端、服务器、发布包或产品行为。
- 不创建虚假的截图、视频或许可证声明。

## 验收标准

- 首屏能在 30 秒内说明产品用途、平台和下载入口。
- 中文主文案完整，英文简介简短且准确。
- 截图/视频位置明确写出用户后续需要准备的内容，不产生失效媒体链接。
- 官网、GitHub Release、隐私政策、用户协议和邮箱链接可访问。
- 不残留面向内部的旧版本发布措辞或真实凭据。
- `git diff --check` 通过，公开仓库敏感信息扫描无命中。
