# 正式发布指南（v1.0.5）

本指南描述正式发行前的安全构建和验收流程。发布准备包位于本机
`F:\code\正式发布准备-v1.0.5`；实际密码、签名文件和生产环境文件不进入 Git。

## Android

1. 复制 `android-app/release-signing.properties.example` 为
   `android-app/release-signing.properties`，在本机填写正式 keystore、alias 和密码。
2. 在本机设置 `WATER_PUBLIC_BASE_URL` 和 `WATER_BOOTSTRAP_TOKEN`，Token 只作为客户端注册配置，不能与管理员密码复用。
3. 执行：

   ```powershell
   cd android-app
   $env:WATER_PUBLIC_BASE_URL = 'https://your-domain.example'
   $env:WATER_BOOTSTRAP_TOKEN = '在本机输入的发布 Token'
   .\scripts\build-formal-release.ps1
   ```

4. 使用 `verify-formal-release.ps1` 检查 APK 包名、版本、API、签名证书和 SHA256。脚本会拒绝 Android Debug 证书。
5. 在真实 Android 14/API 34 及以上设备验证安装、升级、拍照、相册、离线检测、保存和同步。

## Windows

1. 重新生成干净的 PyInstaller 目录，确认 EXE、`_internal`、两个模型和 `release.json` 齐全。
2. 安装 Inno Setup 6，并使用：

   ```powershell
   $env:WATER_PUBLIC_BASE_URL = 'https://your-domain.example'
   $env:WATER_BOOTSTRAP_TOKEN = '在本机输入的发布 Token'
   .\scripts\build-windows-formal-release.ps1 -RequireInstaller
   ```

3. 若要面向公众发布，同时设置 `WATER_SIGNTOOL_PATH`、`WATER_CODESIGN_PFX`、`WATER_CODESIGN_PASSWORD` 并增加 `-RequireCodeSigning`。
4. 在干净 Windows 10/11 x64 环境测试安装、启动、模型加载、结果保存、卸载和重新安装。

## 服务器

在服务器本机设置以下环境变量后执行部署：

```bash
export WATER_PUBLIC_DOMAIN='your-domain.example'
export WATER_CERTBOT_EMAIL='release-owner@example.com'
export WATER_ADMIN_INITIAL_PASSWORD='在服务器本机输入的强密码'
export WATER_BOOTSTRAP_TOKEN='在服务器本机输入的 Token'
sudo -E bash server/deploy/bootstrap.sh /path/to/project/server
```

脚本会自动生成独立 Secret Key，强制生产模式和 HTTPS，并拒绝弱密码、占位值和不安全域名。正式发布 Android 资产前，还要把正式 APK 证书 SHA-256 配置到服务器的 `WATER_ANDROID_SIGNING_CERT_SHA256`。

## 发布前必须补齐

- 项目 `LICENSE` 和第三方依赖/模型权重授权记录。
- 隐私政策、用户协议、运营主体、客服邮箱和数据保留期限。
- Android 应用商店资料或明确的受限分发渠道。
- Windows 代码签名证书和发布后的支持/回滚方案。
